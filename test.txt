## Updating PM2 Processes Later
If you make changes to ecosystem.config.js :
1. Update your processes: pm2 restart ecosystem.config.js
2. Save the new state: pm2 save
3. The service will automatically load the new state on next boot

Why It Got PAUSED
NSSM pauses a service when the application it's running exits too quickly — it thinks the app is misbehaving and pauses instead of endlessly restarting.
This usually means the pm2 resurrect command ran, finished, and exited — which is correct behavior for a one-time command, but NSSM treats it as a crash.
Permanent Fix — Change NSSM to Not Pause on Exit
powershell# Tell NSSM to always restart, never pause
C:\nssm\win64\nssm.exe set PM2-EmailScheduler AppExit Default Restart
C:\nssm\win64\nssm.exe set PM2-EmailScheduler AppRestartDelay 10000

# Then restart the service
Stop-Service PM2-EmailScheduler -Force
Start-Service PM2-EmailScheduler

Verify PM2 Actually Came Back
powershell# After service starts, check PM2
pm2 status

# If PM2 shows no processes, manually resurrect
pm2 resurrect
pm2 status

# Then save again
pm2 save

Check What's in NSSM Logs

type "C:\path\to\project\logs\nssm-error.log"

Fix Everything Now — Run One at a Time in PowerShell Admin
Step 1 — Create the bat file
 Create logs folder if not exists
New-Item -ItemType Directory -Force -Path "C:\Users\lenovo\Desktop\EmailSchedulerMicroService\scheduler\logs"

Open Notepad and save this as C:\Users\lenovo\Desktop\EmailSchedulerMicroService\pm2-start.bat:

bat@echo off
timeout /t 15 /nobreak
call "C:\Users\lenovo\AppData\Roaming\npm\pm2" resurrect
:keepalive
timeout /t 60 /nobreak
pm2 ping >nul 2>&1
if errorlevel 1 (
    pm2 resurrect
)
goto keepalive


Step 2 — Stop NSSM service
powershell
sc stop PM2-EmailScheduler
Wait until you see STOPPED, then:

Step 3 — Point NSSM to bat file
C:\nssm\win64\nssm.exe set PM2-EmailScheduler Application "C:\Windows\System32\cmd.exe"

C:\nssm\win64\nssm.exe set PM2-EmailScheduler AppParameters "/c C:\Users\lenovo\Desktop\EmailSchedulerMicroService\pm2-start.bat"

C:\nssm\win64\nssm.exe set PM2-EmailScheduler AppExit Default Restart

C:\nssm\win64\nssm.exe set PM2-EmailScheduler AppRestartDelay 10000

C:\nssm\win64\nssm.exe set PM2-EmailScheduler AppThrottle 30000

Step 4 — Start PM2 workers manually first
powershell
pm2 restart email-scheduler-workers
pm2 restart log-cleanup-worker
pm2 save

Verify they are online:
pm2 status

Should show online not stopped.

Step 5 — Start NSSM service
sc start PM2-EmailScheduler
Start-Sleep -Seconds 20
Get-Service PM2-EmailScheduler

Should show Running not Paused.

Step 6 — Final verification
pm2 status
